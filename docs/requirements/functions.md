> 사용자 시나리오 기반 기능 정의
> 분류: 공통 / 라이더 / 관리자 / 사장님 / 고객

## 📌 표기 약어

| 약어 | 의미 |
|---|---|

## 1. 공통

### admin-login: 관리자 로그인 (신규)
- **사용자**: 관리자(admin)
- **실행조건**: 랜딩화면 푸터 관리자 버튼 클릭 → 로그인
- **흐름**: 관리자 서비스 화면으로 이동
- **상태변화**: 관리자 로그인 상태
- **예외**: 관리자 로그인 실패

### rider-login: 라이더 로그인 (신규)
- **사용자**: 라이더(rider)
- **실행조건**: 랜딩화면 바디 라이더 버튼 클릭 → 로그인
- **흐름**: 라이더 서비스 화면으로 이동
- **상태변화**: 라이더 로그인 상태
- **예외**: 라이더 로그인 실패

### owner-login: 사장 로그인 (기존)
- **사용자**: 사장(owner)
- **실행조건**: 랜딩화면 바디 입점문의 버튼 클릭 → 로그인
- **흐름**: 사장님 서비스 화면으로 이동

### user-login: 고객 로그인 (기존)
- **사용자**: 고객(user)
- **실행조건**: 랜딩화면 헤더 주문하기 버튼 클릭 → 로그인
- **흐름**: 고객 서비스 화면으로 이동

### admin-logout / owner-logout / user-logout: 로그아웃
- **상태변화**: 각 사용자 로그인 상태 해제, 랜딩 화면으로 이동

### owner-join: 사장 회원가입 (변경)
- **실행조건**: 입점문의 화면에서 회원가입 버튼 클릭
- **흐름**: 사장님 회원가입 화면으로 이동
- **예외**: 사업자 등록 인증 실패, 입력값 누락 시 회원가입 신청 불가
- **비고**: 승인 기준 및 이용약관 확인 필요

### rider-join: 라이더 회원가입 (신규)
- **실행조건**: 랜딩화면 바디 라이더 버튼 클릭 → 회원가입
- **예외**: 면허 인증 실패, 입력값 누락 시 회원가입 신청 불가

## 2. 라이더 (신규)

### rider-order-waiting: 라이더 대기 배달
- **실행조건**: 라이더 배달현황 중 대기 배달 건 상세 화면
- **흐름**: 상세화면 클릭 후 이동 → 배달정보 확인, 수락 시 진행화면 이동
- **예외**: 반려는 이전 화면으로 이동
- **비고**: 실시간 배달 위치 연동 고려

### rider-order-inprogress: 진행 중 배달
- **실행조건**: 라이더 배달 진행중인 건 상세 화면
- **흐름**: 진행 화면으로 이동 → 가게 도착, 배달 완료 버튼 클릭 가능

### rider-order-complete: 배달 완료
- **흐름**: 배달 완료 클릭 시 카메라 전환 → 촬영 → 업로드 → 고객에게 배달 완료 알림

### rider-notice: 라이더 공지사항
- **실행조건**: 라이더 내비게이션 공지사항 버튼
- **흐름**: 공지사항 화면 이동 → 안전수칙, 서비스 태도, 긴급사항 등 명시

### rider-summary: 배달 내역
- **실행조건**: 라이더 내비게이션 배달내역 버튼
- **흐름**:
  - 운행일, 배달건/이동 거리, 배달비합계, 하루치 실수령액 확인
  - 이번 주 정산내역 포함
  - 정산 계좌 확인 및 변경 가능
- **예외**: 계좌 변경 입력값 누락 시 저장 불가
- **비고**: 기간 필터 고려, 실수령액 = 배달료 합계 - 수수료, 정산 정책 기준 자동 산출

### rider-status: 라이더 상태 토글
- **실행조건**: 라이더 내비게이션 상단 토글
- **흐름**: 토글 변경에 따라 배차 여부 변경 (배달중 / 휴식중)

### rider-off: 업무 종료
- **실행조건**: 라이더 내비게이션 하단 버튼
- **흐름**: 종료 모달 표시 → 업무종료 확인 시 로그아웃 / 취소 시 화면 유지
- **예외**: 진행 중 배달 건이 있으면 업무종료 불가 또는 경고

### rider-cs-call: 고객센터 연결
- **실행조건**: 라이더 배달내역 화면 - 정산문의 버튼
- **흐름**: 고객센터 전화번호 안내 모달

## 3. 관리자 (신규)

### admin-dashboard: 대시보드
- **실행조건**: 관리자 서비스 로그인 후 바로 보이는 화면
- **상태변화**: 데이터 카드 정보 확인 (가입자, 가게 등록, 등록 리뷰 수)

### admin-user-management: 회원 관리
- **실행조건**: 관리자 내비게이션 회원관리 버튼
- **흐름**:
  - 특정 회원 검색
  - 가입 승인 또는 탈퇴 관리
  - 회원 정보 열람
  - 페널티 부여
- **예외**: 검색 조건 불일치 또는 입력값 누락 시 검색 불가
- **비고**: 누적 신고 경고 3회부터 / 일시 이용제한 5회 / 이용정지 7회 이상

### admin-store-management: 가게 관리
- **흐름**: 특정 가게 검색, 영업 승인 또는 대기, 가게 정보 열람
- **비고**: 가입 승인/대기/반려 기준

### admin-review-list / admin-review-report: 리뷰 관리
- **흐름**:
  - 전체/신고 리뷰 버튼으로 구분
  - 리뷰 정보(별점) 열람
  - 신고된 리뷰 확인, 활동 제한 설정 및 안내
- **비고**: 활동제한 드롭박스 (작성정지, 영업정지)

### admin-rider-management: 라이더 관리 (실시간 관제)
- **실행조건**: 관리자 내비게이션 배달관리 버튼
- **흐름**:
  - 배달관리 화면 이동
  - 실시간 배달 관제 구역 모니터링
  - 배달 접수 대기 / 배차 대기 / 배달 중 / 배달 완료 건수 확인
  - 라이더 전체공지 작성 모달 (제목, 수신/대상, 공지 내용 입력)
  - 공지 전송 예약 (즉시/예약 전송) 설정
- **상태변화**:
  - 주문 번호, 가게, 진행시간, 라이더 위치 등 배달 상태 확인
  - 문제 발생 시 라이더 연락 가능
  - 공지사항 입력값이 라이더 화면 공지사항에 반영됨
- **예외**: 처리 실패 시 오류 메시지, 입력값 누락 시 등록/수정 실패
- **비고**: 실시간 관제 (지도 API 연동)

### admin-settlement-management: 정산 관리
- **실행조건**: 관리자 내비게이션 정산관리 버튼
- **흐름**:
  - 정산관리 화면 이동
  - 가게/라이더 총 정산예정액, 정산완료, 대기 건수 확인
  - 기간 선택, 대상 구분, 정산 상태 조건 검색
  - 정산 상세 내역에서 특정 주차 선택 시 정산일/수수료/정산 유형 확인
  - 특정 가게/라이더 선택 시 정산 상세 정보 모달
- **상태변화**:
  - 기간/대상/정산 상태 조건에 따라 정산 현황 표시
  - 특정 가게/라이더 선택 시 주문 건별 상세 내역 모달 표시
- **예외**: 조건에 맞는 결과 없을 경우 안내 문구
- **비고**: 정산은 주 단위 기준 집계, 수수료 및 정산 기준은 정책에 따라 표시

### admin-cs-management: 고객 지원 관리
- **실행조건**: 관리자 내비게이션 고객 지원 버튼
- **흐름**:
  - 고객 지원 관리 화면 이동
  - 문의 현황 카드 확인 (총 문의, 자동 응대 완료, 미처리 문의)
  - 챗봇 미처리 문의 목록 및 상세 확인
  - 미처리 문의 상세 → 답변 작성 후 전송 완료
  - FAQ 카테고리 탭 필터 (결제/배달/환불취소/회원)
  - FAQ 등록/수정/삭제
- **상태변화**:
  - 챗봇 문의 → 자동 응대 완료/미처리 문의로 분기
  - 챗봇 미처리 문의 답변 전송 완료 시 처리 완료 표시
  - FAQ 수정 시 결과 반영
- **예외**: 챗봇 응답 실패 시 미처리 문의로 자동 전환, 문의 0건 시 0건 표시
- **비고**: 챗봇 실시간 문의 건수 데이터 자동 갱신 협의 필요

### admin-policy-list: 정책 목록
- **흐름**: 등록된 정책 목록 확인, 정책 종류(취소/환불/개인정보) 필터
- **비고**: 정책 종류 = cancel, refund, privacy 등. 친환경 점수 기준 정할 것.

### admin-policy-create: 정책 등록
- **실행조건**: 정책관리 화면에서 등록 버튼 클릭
- **흐름**: 정책 작성 화면 → 종류/제목/내용 입력 → 등록
- **상태변화**: policies 테이블에 신규 정책 저장 (is_active=true, version=1)
- **예외**: 필수값 누락 시 등록 실패, 같은 종류의 활성 정책 있으면 경고

### admin-policy-edit: 정책 수정 (버전 관리)
- **흐름**: 정책 목록 수정 버튼 → 수정 화면 → 내용 수정 → 저장
- **상태변화**:
  - 기존 버전 is_active = false
  - 새 버전 생성 (version + 1, is_active = true)
- **예외**: 수정 시 이전 버전 이력 유지
- **비고**: 버전 관리로 정책 변경 추적 가능

### admin-policy-delete: 정책 비활성화
- **흐름**: 비활성화 확인 모달 → 확인
- **상태변화**: 해당 정책 is_active = false
- **예외**: 이미 동의한 사용자 기록은 유지
- **비고**: 물리 삭제 아닌 **논리 삭제**

## 4. 사장님

### owner-menu-option: 메뉴 옵션 관리 (신규)
- **실행조건**: 사장님이 메뉴 옵션추가 버튼 클릭
- **상태변화**: 옵션 추가 (맵기, 사리추가, 사이즈 등)
- **비고**: 메뉴 카테고리처럼 옵션 카테고리

### owner-menu-alert: 실시간 주문 알림 (신규)
- **실행조건**: 고객 주문 시 즉시 사장님 화면에 표시
- **상태변화**: 신규 주문 실시간 생성, 시각적 효과
- **비고**: SSE

### owner-review-rating: 고객 리뷰 별점 통계 (신규)
- **실행조건**: 사장 서비스 내비게이션 리뷰관리 버튼
- **흐름**: 평균 별점 확인, 악성 리뷰일 경우 신고
- **비고**: 차트 라이브러리

### owner-orderlist: 주문 현황 (기존)
- **실행조건**: 사장님 서비스 로그인 후 메인 화면
- **흐름**: 주문 상태 건수 확인 → 신규 주문 목록 및 관리
- **상태변화**: 총 주문 수, 주문 수락 대기, 배달 완료, 취소 건수 등 일일 주문 현황 표시

### owner-order-status: 주문 상태 관리 (기존)
- **실행조건**: 신규 주문 목록 - 주문 상태 버튼 클릭
- **흐름**: 신규 주문 확인, 상태 변경 처리
- **상태변화**: 주문 수락 / 조리 중 / 배달 시작 / 배달 완료 / 주문 취소

### owner-store-select: 가게 선택 (기존)
- **흐름**: 드롭다운 목록에서 관리할 가게 선택 → 해당 가게 관리 화면 이동

### owner-store-sales: 매출 관리 (기존)
- **실행조건**: 사장님 내비게이션 매출 관리 버튼
- **흐름**: 매출 현황 → 기간별 그래프, 상품 매출 순위
- **예외**: 상품 매출 데이터 없을 시 안내 문구

### owner-store-menu: 메뉴 관리 (기존)
- **실행조건**: 사장님 내비게이션 메뉴 관리 버튼
- **흐름**: 등록 메뉴 확인 → 메뉴 추가/수정/삭제, 카테고리 관리

### owner-store-menu-add / edit / delete: 메뉴 CRUD (기존)
- **흐름**: 모달 생성 → 메뉴 정보 입력/수정/삭제 → 메뉴 목록 반영
- **예외**: 입력값 누락 시 저장/수정 불가, 취소 시 모달 종료

### owner-store-menu-category: 카테고리 관리 (기존)
- **흐름**: 카테고리 추가/수정/삭제 → 메뉴 목록 반영
- **예외**: 카테고리 삭제 시 포함 메뉴 모두 삭제, 삭제 전 경고 메시지

### owner-store-management: 가게 관리 (기존)
- **실행조건**: 사장님 내비게이션 가게 관리 버튼
- **흐름**: 기존 가게 정보 확인 → 상호명/주소/전화/사업자번호/사진 수정, 가게 삭제
- **예외**: 가게 삭제 시 경고
- **비고**: 사업자번호 변경 시 추가 인증 여부 확인 필요

### owner-store-status: 가게 운영 상태 수정 (기존)
- **흐름**: 영업 상태/시간/휴무일/최소 주문 금액/공지사항 수정

### owner-store-add: 가게 추가 (기존)
- **실행조건**: 사장님 내비게이션 가게 추가 버튼
- **흐름**: 가게 등록 화면 → 기본 정보 입력 및 저장
- **예외**: 입력값 누락 시 추가 불가, 취소 시 주문 관리 화면으로 이동

## 5. 고객

### user-store-menu-search: 가게 내 메뉴 검색 (신규)
- **실행조건**: 고객이 가게 내에서 메뉴 검색
- **흐름**: 가게 상세 → 메뉴 검색
- **예외**: 검색어 띄어쓰기 오류 시 검색 불가
- **비고**: 작은 검색창

### user-cancel: 주문 취소 (신규)
- **실행조건**: 가게 주문 수락 전 취소버튼 클릭
- **흐름**: 주문 취소 사유 선택창
- **상태변화**: 주문상태 대기 → 취소
- **예외**: 가게 승인 후 버튼 비활성화

### user-refund: 결제 취소 (신규)
- **실행조건**: 주문 취소 상태로 변경될 때
- **상태변화**: 결제상태 결제완료 → 환불완료

### user-cart-badge: 장바구니 배지 (신규)
- **실행조건**: 장바구니 담기
- **흐름**: 메뉴 담기 → 하단 장바구니 아이콘 위 숫자 카운트 업데이트
- **예외**: 장바구니 비어있을 때 숫자 숨김
- **비고**: Pinia/Vuex 상태관리

### user-find-auth: 계정 찾기 (신규)
- **실행조건**: 로그인 화면 하단 버튼
- **흐름**: 이름/연락처 인증 → 아이디 노출 또는 비밀번호 재설정

### user-chatbot: 1:1 문의 챗봇 (신규)
- **실행조건**: 서비스 하단 챗봇 플로팅 버튼
- **흐름**:
  - 챗봇 아이콘 클릭 → 대화창 활성화
  - 시나리오 선택 또는 텍스트 입력
  - 챗봇 해결 불가 시 문의 남기기로 이동

### user-membership: 펫 멤버십 (신규)
- **실행조건**: 주문 또는 펫화면 접속
- **흐름**: 주문/접속으로 성장치 상승 → 레벨업 시 포인트 지급
- **상태변화**: 성장치 상승하다가 일정치 이상에서 레벨업
- **비고**: 회원 등급 조회

### user-order-status: 고객 배달 현황 (신규)
- **실행조건**: 고객 마이페이지 - 주문내역 상세 정보 버튼
- **흐름**: 상세 정보 화면 → 실시간 배달 현황 (새로고침 X)

### user-review-post: 리뷰 작성 (기존)
- **실행조건**: 주문완료 화면 - 리뷰작성 버튼
- **흐름**: 별점, 사진과 함께 리뷰 작성

### user-store-list: 가게 목록 (기존)
- **실행조건**: 홈 화면 - 카테고리 카드, 메뉴 전체보기 클릭
- **흐름**: 조건에 맞는 가게 목록, 정렬 변경, 카테고리 바 이동

### user-store-search: 가게/메뉴 검색 (기존)
- **실행조건**: 상단 검색창
- **예외**: 결과 없을 시 안내 문구, 다시 검색하기 클릭 시 홈 이동

### user-store-detail: 가게 상세 (기존)
- **흐름**: 메뉴, 가게 정보, 리뷰 표시

### user-cart: 장바구니 (기존)
- **실행조건**: 홈 장바구니 아이콘 또는 메뉴 담기 클릭
- **흐름**: 메뉴 목록 확인, 수량 변경/삭제, 주문하기, 전체 비우기
- **상태변화**: 메뉴/수량/금액/배달팁/총 주문 금액 표시, 전체 비우기 시 초기화

### user-payment: 결제 (기존)
- **실행조건**: 메뉴 담고 주문하기 클릭
- **흐름**: 결제하기 화면
- **상태변화**:
  - 배달 정보, 휴대폰 번호, 요청사항, 결제 수단, 약관동의, 쿠폰 선택, 친환경 선택, 총 결제금액 표시
  - 친환경 선택 시 친환경 점수 자동 적립
- **예외**: 결제 오류 시 안내 문구
- **비고**: 토스페이먼츠 API 연동

### user-order-detail: 주문 상세 조회 (기존)
- **실행조건**: 마이페이지 - 주문 내역 - 특정 주문 상세

### user-store-favorite: 가게 찜 (기존)
- **실행조건**: 고객 내비게이션 - 찜 / 가게 상세 - 찜 클릭
- **상태변화**: 찜 설정 가게 목록 조회/해제
- **예외**: 찜 목록 없을 시 안내 문구, 가게 둘러보기 클릭 시 홈 이동

### user-nearby: 주변 찾기 (기존)
- **실행조건**: 고객 내비게이션 - 주변 찾기
- **흐름**: 설정 주소 기준 3km 이내 가게 거리순
- **예외**: 3km 이내 없을 시 안내 문구, 주소 설정하러 가기 클릭

### user-cs: 고객센터 (기존)
- **실행조건**: 고객 내비게이션 - 고객센터
- **흐름**: 자주 묻는 질문 / 상담원 연결 클릭 시 챗봇 이동
- **비고**: 챗봇 연결

### user-mypage: 마이페이지 (기존)
- **흐름**: 내 정보 수정/주소관리/쿠폰/주문 내역/리뷰 관리/내 펫 화면 표시

### user-myinfo: 내 정보 수정 (기존)
- **흐름**: 기본 정보 수정, 누적 친환경 점수 표시
- **예외**: 입력값 오류/누락 시 수정 불가

### user-address: 주소 관리 (기존)
- **흐름**: 기본 주소 설정, 신규 주소 추가
- **비고**: 주소 검색 API

### user-orderlist: 주문 내역 (기존)
- **흐름**: 고객 주문 내역 조회

### user-coupon: 쿠폰함 (신규)
- **흐름**: 보유 쿠폰 조회, 결제 단계에서 쿠폰 선택 적용

### user-review-management: 리뷰 관리 (기존)
- **흐름**: 작성 리뷰 내역 수정/삭제

### user-event: 이벤트 참여 (기존)
- **실행조건**: 홈 화면 - 이벤트 배너
- **흐름**: 룰렛 또는 미니게임 참여 → 쿠폰 지급
- **비고**: 참여 횟수 초과 시 참여 불가 고려

### user-policy-view: 정책 열람 (신규)
- **실행조건**: 서비스 하단 이용약관/정책 링크, 또는 주문취소 시 환불정책 팝업
- **흐름**: 활성(is_active=true) 정책 표시
- **비고**: 회원가입 시 필수 정책 동의 체크

### user-policy-agree: 정책 동의 (신규)
- **실행조건**: 회원가입 화면 정책 동의 체크박스
- **흐름**: 필수 정책 동의 체크 → 가입 진행
- **상태변화**: policy_agreements에 동의 이력 저장
- **예외**: 필수 정책 미동의 시 가입 불가
