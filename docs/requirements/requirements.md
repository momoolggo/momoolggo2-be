## 📌 우선순위 정의

| 우선순위 | 의미 |
|---|---|
| **상** | MVP 필수 기능, Phase 5에서 최우선 |
| **중** | 핵심 보조 기능 |

## 1. 공통 (9개)

### REQ-COM-001: 고객 회원가입
| 항목 | 내용 |
|---|---|
| **관련 API** | No.5 회원가입, No.53 정책 동의 |
| **분류** | 기능 |
| **우선순위** | 상 |
| **상세** | 고객이 아이디/비밀번호/이름/생년월일/성별/연락처를 입력해 계정 생성. 회원가입 시 필수 정책(개인정보, 이용약관) 동의 필수. |
| **입력** | userId, userPw, name, birth, gender, tel, policyIds[] |
| **출력** | 회원가입 성공 메시지, 계정 생성 |
| **선행** | 아이디 중복확인 통과 |
| **후행** | 고객 로그인 가능 |
| **예외** | 중복 시 가입 불가, 입력값 누락 시 가입 불가, 필수 정책 미동의 시 가입 불가 |

  - 각 서비스 독립 MySQL 스키마 사용 (`my_mmg_*`)
- **비고**: Spring Boot + Spring Cloud Gateway, OpenFeign 서비스 간 통신

### REQ-NFR-003: 실시간 통신 (SSE)
- **우선순위**: 중

- **우선순위**: 중

  - 고객 1:1 문의 챗봇 = Gemini API 자동 응답
